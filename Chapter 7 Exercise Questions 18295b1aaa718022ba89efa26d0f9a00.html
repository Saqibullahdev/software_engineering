7.6. Object-Oriented Design for Systems

  • Question: Identify possible objects in the following systems and develop an object-oriented design for them. Make reasonable assumptions about the systems.

Answer:

  1. Group Diary and Time Management System:
    • Objects:
      • User: Represents a co-worker with a personal diary.
      • Appointment: Represents a meeting or appointment.
      • Diary: Manages appointments for a user.
      • Scheduler: Finds common slots and schedules appointments.
      • Notification: Sends reminders or updates to users.
    • Design:
      • The Scheduler interacts with multiple Diary objects to find common slots.
      • If no slots are available, the Scheduler interacts with the User to rearrange their diary.
  1. Automated Filling Station (Gas Station):
    • Objects:
      • Pump: Represents a fuel pump.
      • CardReader: Reads and verifies credit cards.
      • CreditCompany: Verifies the card and sets a fuel limit.
      • FuelTank: Manages fuel levels.
      • Transaction: Records the fuel purchase and debits the card.
    • Design:
      • The CardReader communicates with the CreditCompany to verify the card.
      • The Pump dispenses fuel and updates the Transaction object.
      • If the card is invalid, the CardReader returns it without dispensing fuel.

7.9. Importance of Configuration Management

  • Question: Using examples, explain why configuration management is important when a team of people are developing a software product.

Answer:

  • Configuration Management (CM) is crucial for team-based software development because:
    1. Version Control: It keeps track of different versions of software components, ensuring that everyone is working on the latest version.
      • Example: If two developers are working on the same file, CM tools like Git prevent conflicts by merging changes.
    1. Collaboration: It allows multiple developers to work on the same project without overwriting each other’s work.
      • Example: A team developing a mobile app can use CM to manage code, assets, and documentation.
    1. Traceability: It provides a history of changes, making it easier to identify when and why a bug was introduced.
      • Example: If a bug is found, the team can trace it back to a specific change using CM tools.
    1. Build Management: It automates the process of compiling and linking components to create a working system.
      • Example: A CM system can automatically build and test the software whenever a new change is committed.
    1. Problem Tracking: It helps manage bug reports and ensures that issues are resolved systematically.
      • Example: A CM system like Jira can track bugs and assign them to developers.

7.10. Open Source for Customized Software

  • Question: A small company has developed a specialized product that it configures for each customer. New customers usually have specific requirements, and they pay for these to be developed. The company has an opportunity to bid for a new contract, which would more than double its customer base. The new customer also wishes to have some involvement in the configuration of the system. Explain why, in these circumstances, it might be a good idea for the company to make the software open source.

Answer:

  • Reasons to Make the Software Open Source:
    1. Customer Involvement: The new customer wants to be involved in the configuration. Open source allows them to customize the software to their needs without relying entirely on the company.
    1. Community Contributions: Open source can attract a community of developers who can contribute to the software, reducing the development burden on the company.
    1. Faster Development: With more contributors, new features and configurations can be developed faster, helping the company meet the demands of a larger customer base.
    1. Cost Savings: The company can save on development costs by leveraging community contributions.
    1. Transparency: Open source builds trust with customers, as they can see and verify the code.
    1. Market Expansion: Open source can increase the software’s visibility and attract more customers.
